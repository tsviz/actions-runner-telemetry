## File Naming Convention

Reports follow this naming pattern:
```
telemetry-report.{runner}-{visibility}-{scenario}.md
```

- **runner**: The runner label (e.g., `ubuntu-latest`, `linux-8-core`, `self-hosted-linux`)
- **visibility**: Repository visibility (`public` or `private`)
- **scenario**: Workload pattern (e.g., `underutilized`, `overutilized`, `balanced`)

## Viewing Reports

When using the action in your workflows, reports are automatically displayed in the **GitHub Actions Job Summary** on your workflow run page - no extra steps needed!

These markdown files also render well when viewed:
1. **On GitHub** - Mermaid charts render natively
2. **In VS Code** - With the Markdown Preview Enhanced extension
3. **In any markdown viewer** that supports Mermaid diagrams

## Generating Your Own

To generate reports for your workflows, add the action to your workflow:

```yaml
- uses: tsviz/actions-runner-telemetry
```

See the [main README](../../README.md) for full usage instructions.
